# PMtool 开发路线图

## Phase 0: 方向收口

- 明确产品是 AI PM 引擎，而不是传统 PM SaaS
- 固化“企业微信 + 腾讯表格 + Agent 后端”的产品形态
- 统一 README、产品文档、调试方式和验收口径

## Phase 1: 核心主链路

- 接入企业微信消息入口
- 支持意图识别与项目路由
- 打通需求拆解 -> 审核 -> 分配的主链路
- 建立基础数据模型与队列机制

## Phase 2: 项目接入与运维

- 提供 setup/dashboard 配置页
- 支持 workspace 级 adapter 配置
- 支持项目级群聊和 Webhook 配置
- 增加快速测试、链路测试、手动触发操作

## Phase 3: 自动化管理动作

- 风险扫描
- 周报生成
- 产能快照
- 变更评估与执行
- 项目复盘

## Phase 4: 稳定性与可用性

- 修复链路中的行为漂移
- 补齐集成测试与真实环境验证
- 清理占位实现
- 优化运行日志、错误提示、部署与回滚流程

## 当前优先级

1. 主链路稳定性
2. Setup / Dashboard 真实性
3. 腾讯表格真实联调
4. 占位模块清理
5. 自动化管理动作质量提升

## 暂缓事项

- 完整账号体系
- 独立任务看板前台
- 传统需求/迭代 Web 页面
- 多组织权限系统
