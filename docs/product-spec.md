# PM Tool MVP 产品说明

## 1. 产品定位

PM Tool 是一个服务于产品经理和小型项目团队的协作工具，目标是把需求、任务、迭代和进度视图集中到一个系统中，降低信息分散和推进成本。

## 2. 目标用户

- 产品经理
- 项目经理 / 项目负责人
- 开发成员
- 测试成员

## 3. 核心问题

当前团队常见问题包括：

- 需求文档、任务分配、版本计划分散在多个平台
- 项目状态依赖口头同步，容易遗漏
- 缺少适合小团队快速落地的轻量 PM 工具

## 4. MVP 核心模块

### 4.1 账户与权限

- 用户注册 / 登录
- 基础个人信息
- 简单角色区分

### 4.2 项目管理

- 创建项目
- 编辑项目基础信息
- 项目列表与项目详情

### 4.3 需求管理

- 创建需求
- 设置优先级、状态、负责人
- 需求与项目关联

### 4.4 任务管理

- 从需求拆解任务
- 任务状态流转
- 任务负责人、截止时间、备注

### 4.5 看板视图

- 按状态展示任务
- 支持拖拽变更状态

### 4.6 版本 / 迭代管理

- 创建迭代
- 绑定需求与任务
- 查看迭代进度

### 4.7 数据概览

- 项目任务总数
- 各状态任务数量
- 即将到期任务

## 5. MVP 之外暂不纳入

为了控制复杂度，以下能力先不进入第一阶段：

- 甘特图
- 工时统计
- 审批流
- 多组织 / 多租户
- 深度消息通知系统
- 自定义工作流引擎

## 6. 核心页面建议

- 登录页
- 项目列表页
- 项目详情页
- 需求列表页
- 任务看板页
- 迭代管理页
- 数据概览页

## 7. 成功标准

如果 MVP 能达到以下目标，就说明方向成立：

- 团队可以完整录入一个项目
- 一个需求能顺利拆分成任务并推进到完成
- 项目负责人能通过看板快速掌握进度
- 产品经理能查看版本范围和关键风险
