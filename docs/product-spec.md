# PMtool 产品说明

## 1. 产品定位

PMtool 是一个面向小团队的 AI 项目管理引擎。

它不追求替代完整的 Web PM SaaS，而是把团队已经在用的沟通和协作介质串起来：

- 企业微信：作为操作入口
- 腾讯智能表格：作为结果展示与协作面板
- PMtool 后端：负责 Agent 编排、任务拆解、审核、分配和同步

## 2. 目标用户

- 产品经理 / 项目经理
- 小型研发团队负责人
- 需要在群聊中快速推进需求的执行团队

## 3. 核心问题

- 需求讨论发生在群里，但后续拆解、分配、跟进分散在多处
- 小团队没有专职 PMO，任务推进严重依赖人工盯办
- 风险、变更、周报等管理动作成本高，经常滞后
- 传统 PM 工具太重，难以在现有协作习惯中自然落地

## 4. 当前 MVP 范围

### 4.1 Workspace 与适配器配置

- 配置企业名称
- 配置企业微信 BOT 参数
- 配置默认 AI 模型与 Provider API Key
- 配置项目级群聊与腾讯表格 Webhook

### 4.2 项目接入

- 创建项目
- 绑定项目群 / 管理群
- 配置任务表、排期表、产能表、风险表、变更表

### 4.3 群聊驱动的需求处理

- 在企业微信群内 `@助手`
- 识别需求拆解、周报、风险扫描、产能评估、变更评估、复盘等意图
- 将需求路由到对应 Agent 处理

### 4.4 Agent 主链路

- 中枢：识别意图并路由
- 中书省：将需求拆成任务，或识别为排期模板实例化
- 门下省：审核拆解结果，必要时否决并给出修正建议
- 尚书省：分配负责人、执行变更后的级联更新

### 4.5 自动化管理动作

- 每日产能快照
- 每日风险扫描
- 每周周报生成
- 项目复盘生成

### 4.6 腾讯智能表格同步

- 任务同步
- 管线阶段同步
- 风险同步
- 产能快照同步

## 5. 当前不纳入 MVP 的内容

- 独立的登录/注册系统
- 完整 Web 看板、需求页、迭代页
- 多租户组织体系
- 复杂审批流
- 通用工作流设计器
- 重型 BI 报表系统

## 6. 成功标准

- 团队能完成一个项目的接入配置
- 产品经理能在群里发起需求分析并拿到结构化任务
- 审核、分配、风险扫描可以跑通一条闭环
- 腾讯表格中能看到关键执行结果
- 团队负责人能收到周报、风险、变更、复盘等管理输出

## 7. 当前阶段判断

当前阶段不是“传统 PM 工具前台 MVP”，而是“AI PM 引擎 Alpha”：

- 主链路已具备可联调能力
- Setup / Dashboard 已具备基础运维能力
- 多个 Agent 已经落地
- 仍需继续补齐稳定性、测试基线和部分占位模块
